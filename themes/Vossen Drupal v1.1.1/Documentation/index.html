General Information

Built on the Bootstrap 3.x framework, Vossen is a one-page multipurpose corporate/creative Drupal 7 theme. Vossen has been built with ease of use and customizability in mind, allowing you to build a Drupal website with many functions and great possibilities. The theme has been tested extensively and performs well on a broad range of screen sizes and devices.

Vossen for Drupal has been built by Created by Cocoon.

The theme's download package contains the following directories:

  • Documentation - Help files for theme installation, use and customization
  • Demo Installation - The full Drupal installation package, complete with Vossen and all modules installed and pre-configured
  • Existing Installations - The packages required to install Vossen on an existing Drupal instance
  • Licensing - Licensing details pertaining to the theme


NOTE We highly recommend that you install Drupal using our Demo Installation package. This will make the setup of your website easier, as everything will be pre-configured.

Demo Installation

These instructions will show you how to install a fresh instance of Drupal 7, complete with Vossen installed, all modules pre-configured, and extensive demo content with placeholder images.


Step 1. Create Database

You must create a new database for your Drupal installation. The database can be named however you wish. The database can be created via phpMyAdmin, as shown below. (You may also need to create a new database user, unless you plan on using an existing one.)


Step 2. Populate Database

Import the SQL.zip file, named vossen_demo.sql.zip (located in the 'Demo Installation' folder in your theme download package), into your new database. This can be accomplished via phpMyAdmin, as shown below.


Step 3. Upload files

Upload the files.zip archive (located in the 'Demo Installation' folder) to your web server, and extract it using either cPanel, another administration interface, or command line. Alternatively, you can manually upload the entire contents of the uncompressed 'files' folder to your web server. Ensure that the .htaccess file is also transferred, as this may be hidden in your computer's file browser. We recommend using an FTP client, such as FileZilla.


Step 4. Configure settings.php

You must now configure Drupal's settings.php file to reflect your new database credentials, so that Drupal can connect to the database. Open sites/default/settings.php on your web server, and around line 220, edit the database name, user, and password to match the new database credentials you set up in Step 1.


Step 5: Complete! Login!

The full demo installation is now complete. You can now login to your new Drupal website with the following administrator credentials.


USERNAME: admin
PASSWORD: admin



NOTE Vossen ships with two themes: Vossen and Vossen Essentials. Please keep BOTH themes enabled, as Vossen Essentials provides some core functionality for the main Vossen theme. If Vossen Essentials is disabled, some functionality may be lost.

For Existing Installations

Vossen theme can be installed on existing Drupal installations, though this may require more effort to setup and configure.
Ensure that you have a fresh, working backup of your existing Drupal instance (files AND database) before proceeding.


Step 1. Upload theme folders & enable themes

You will find two themes located in the 'Existing Installations/themes' folder, Vossen and Vossen Essentials. Upload both themes to your Drupal themes directory, located at sites/all/themes. Once uploaded, both themes must be enabled at admin/appearance. The Vossen theme must be set as the default theme, though Vossen Essentials must still be enabled.


NOTE Vossen ships with two themes: Vossen and Vossen Essentials. Please keep BOTH themes enabled, as Vossen Essentials provides some core functionality for the main Vossen theme. If Vossen Essentials is disabled or missing, some functionality may be lost.


You must now set the appropriate Theme Settings so that Vossen performs correctly.


Step 2. Upload module folders & enable modules

The modules you must install are located in the 'Existing Installations/modules' folder. Upload the modules to your Drupal modules directory, located at sites/all/modules. Once uploaded, the modules must be enabled via the Drupal Modules page, located at admin/modules. Ensure that you have enabled all modules.


NOTE Vossen utilises a variety of contrib modules. There is a possibility that you may already have some of these modules installed on your Drupal site. If this is the case, only upload the modules which you are not already using to your sites/all/modules folder.


Step 3. Import Taxonomy Vocabularies & Content Types

In order to recreate the layouts and features shown in the demo, such as the portfolio, sliders, client lists and multiple blog layouts, you will need to import the Taxonomy Vocabularies and Content Types which have been setup to work with Vossen.


  • 1. Ensure that the bundles_copy module is installed and enabled. This will allow you to import Taxonomy Vocabularies and Content Types into your existing Drupal installation.
  • 2. Navigate to 'admin/structure/taxonomy/import'. In the textarea provided, paste the entire contents of the 'taxonomy.txt' file located in the 'Existing Installations/import' folder in your theme download package.
  • 3. Navigate to 'admin/structure/types/import'. In the textarea provided, paste the entire contents of the 'types.txt' file located in the 'Existing Installation/import' folder in your theme download package.

Step 4. Import Views

Vossen uses Views to deliver feature-rich pages with lots of dynamic content. All .txt files referenced below are located in the 'Existing Installations/import/views' folder in your download package. In the following order:


  • 1. Navigate to 'admin/structure/views/import'.
    In the textarea provided, paste the entire contents of the 'view-blog.txt' file. Press import.

  • 2. Navigate to 'admin/structure/views/import'.
    In the textarea provided, paste the entire contents of the 'view-clientlist.txt' file. Press import.

  • 3. Navigate to 'admin/structure/views/import'.
    In the textarea provided, paste the entire contents of the 'view-contentslider.txt' file. Press import.

  • 4. Navigate to 'admin/structure/views/import'.
    In the textarea provided, paste the entire contents of the 'view-portfoliocategories.txt' file. Press import.

  • 5. Navigate to 'admin/structure/views/import'.
    In the textarea provided, paste the entire contents of the 'view-portfolio.txt' file. Press import.

  • 6. Navigate to 'admin/structure/views/import'.
    In the textarea provided, paste the entire contents of the 'view-pricingtable.txt' file. Press import.

  • 7. Navigate to 'admin/structure/views/import'.
    In the textarea provided, paste the entire contents of the 'view-revolutionslider.txt' file. Press import.

  • 8. Navigate to 'admin/structure/views/import'.
    In the textarea provided, paste the entire contents of the 'view-taxonomy.txt' file. Press import.

  • 9. Navigate to 'admin/structure/views/import'.
    In the textarea provided, paste the entire contents of the 'view-team.txt' file. Press import.

  • 10. Navigate to 'admin/structure/views/import'.
    In the textarea provided, paste the entire contents of the 'view-sitetweets.txt' file. Press import.


Step 5. Complete!

The Vossen theme and module installation is now complete.

Theme Settings

Vossen's Theme Settings are important to your website's functionality, as they dictate how the theme will run. You can access the Theme Settings at admin/appearance/settings/vossen.


Theme Skin

The Theme Skin settings allow you to change the appearance of Vossen. You can select a default skin out of 15 presets, Green is the default. You can also make further modifications to the theme by including Custom CSS styling in the textarea provided. Any CSS you specify will load on every page.



Theme Features

The Theme Features settings provide more functional options. You can toggle the homepage preloader, scroll to top button, and fixed header on or off.



Map Settings

The Map Settings options provide integration with Google Maps. The map will be displayed in the contact section on the homepage. You may also specify a custom path for the map marker.



Simplenews Settings

The Simplenews Settings options provide AJAX integration with Simplenews blocks. By default, the entire page will refresh when a user subscribes to a newsletter via a Simplenews block. By specifying the ID of the newsletter category, we can integrate an AJAX submission for that particular block, which allows the form to submit elegantly without refreshing the page. The ID for your newsletter category can be found in the URL when you edit the category, e.g. 'admin/config/services/simplenews/categories/1/edit'. Visit 'admin/config/services/simplenews' to view your newsletter categories or to edit them. To remove AJAX submission from all Simplenews forms simply remove the number values from the text fields in the image shown below.



jQuery Update

Ensure that jQuery version 1.11 is selected from the jQuery Update dropdown menu.

Creating Portfolio Items

There is a content type available specifically for portfolio items. Any nodes created with the content type 'Portfolio Item' will appear in the homepage portfolio. Visit Admin > Content > Add Content > Portfolio Item (node/add/portfolio-item) to create a new portfolio item. The Portfolio Item content type has a variety of fields, as follows. These are example fields of how you may layout your Portfolio Items.

  • Title (title of the project)
  • Subtitle (a brief description of the portfolio item)
  • Short project description (a short description of the project, visible on the homepage portfolio display)
  • Medium project description (an overview of the project, displayed beside the long description - great for any additional information)
  • Long project description (the main body - all about the project)
  • Portfolio category (the category under which the item will be listed - these are based on the Portfolio Categories taxonomy terms)
  • URL (provides a direct link to the work - e.g. the project's website)
  • Date (when the project was undertaken/completed)
  • Client (who the project was completed for)
  • Cover image (the image that will appear on the homepage)
  • Lightbox image (the image that will appear in a lightbox-type display that the user may initiate on the homepage)
  • Post type -> Image/Image Slider/Video (the media attached to the Portfolio Item may be one of three types, as with blog posts)
  • Tags (tag the portfolio item with taxonomy terms)

Newsletter

The Simplenews contrib module for Drupal provides the newsletter subscription functionality. You can add, edit, or remove newsletter categories at admin/config/services/simplenews. Vossen is designed to work with two newsletter categories,

  • a website newsletter,
  • and a launch list.
Simplenews creates subscription blocks for each newsletter, where users can enter their email address to subscribe. By default, the website newsletter subscription block appears in the page footer, and the launch list subscription block appears in the 'Hero Landing Page' region.

The Simplenews subscription blocks work by refreshing the page when the user submits the subscription form, and displaying a corresponding success or error message (if the email address was incorrect, for example). However, Vossen is capable of submitting the subscription forms using AJAX, avoiding a page reload when the user submits the form. Instead, a small success/error message is displayed beside the form when the user submits it.



To enable AJAX submission for a particular subscription block form, enter the ID of the corresponding Newsletter Category on the Theme Settings page, at Admin > Appearance > Vossen (admin/appearance/settings/vossen), as shown below.
